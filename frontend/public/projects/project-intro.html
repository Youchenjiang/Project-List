專案概述

Project List 是一個集合多種互動效果與功能的專案庫，旨在展示現代網頁前端開發的各種技術與設計模式。它既是初學者的學習資源，也是開發者的靈感來源。

專案最大的特色在於：每個子專案都可以獨立運行，即使與主框架分離。這意味著你可以單獨取用任何一個效果或功能，並將其整合到自己的專案中。

功能特色

  • 獨立運行: 每個專案都可以脫離主框架獨立執行，方便學習特定功能
  • 模組化設計: 所有功能都以模組化方式開發，便於理解和整合
  • 互動式效果: 豐富的互動效果展示，包括動畫、過渡和響應式設計
  • 資料視覺化: 使用 Echarts 進行資料視覺化，展示複雜數據的圖形化呈現
  • AI 整合: 展示與 OpenAI API 的整合，呈現人工智能應用場景
  • 自適應界面: 所有專案均設計為在不同尺寸的裝置上展示良好

專案結構

Project-List/
├── frontend/              # 前端React應用
│   ├── public/            # 靜態檔案和資源
│   │   ├── assets/        # 靜態資源
│   │   │   ├── images/    # 圖片
│   │   │   └── icons/     # 圖示
│   │   ├── projects/      # 專案範例(每個可獨立執行)
│   │   │   ├── Effect-Download/ # 下載進度效果
│   │   │   ├── Effect-Ripple/   # 波紋動畫效果
│   │   │   ├── Execute-OpenAI/  # OpenAI API 整合
│   │   │   └── Visual Map/      # 地圖視覺化
│   ├── src/               # 原始碼
│   │   ├── components/    # 可複用UI元件
│   │   ├── context/       # React狀態管理上下文
│   │   ├── styles/        # 按元件組織的CSS樣式
│   │   └── data/          # 資料檔案
├── backend/               # 後端服務
│   ├── go.mod             # Go模組依賴項
│   └── main.go            # 主Go伺服器檔案

這種結構設計使得每個專案都能夠：

  • 包含完整的獨立HTML、CSS和JavaScript檔案
  • 不依賴外部框架可直接運行
  • 容易被提取到任何其他專案中使用

安裝與使用

Project List 提供了兩種使用方式：直接訪問獨立專案，或者設置完整開發環境。

方法一：直接訪問獨立專案

每個專案都設計為可獨立執行，只需在瀏覽器中開啟對應的 HTML 檔案即可：

  • 點擊上方的「查看專案」按鈕
  • 或直接瀏覽 frontend/public/projects/ 目錄下的專案

方法二：設置完整開發環境

  1. 複製倉庫
    git clone https://github.com/Youchenjiang/Project-List.git
    cd Project-List
  2. 安裝依賴並啟動應用
    cd frontend
    npm install
    npm start
  3. 瀏覽應用
    開啟瀏覽器，訪問 http://localhost:3000
  4. （可選）運行後端服務
    cd ../backend
    go mod download
    go run main.go
